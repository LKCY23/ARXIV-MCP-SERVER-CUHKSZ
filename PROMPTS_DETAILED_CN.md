# Prompts 详细内容中文说明

本文档详细说明所有 Prompts 的具体内容，告诉你每个 Prompt 到底在指导 AI 做什么。

---

## 1. `deep-paper-analysis` - 深度论文分析 Prompt

### 📋 Prompt 的核心任务
指导 AI 作为一个研究助手，深入分析一篇 arXiv 论文。

### 🔧 可用工具说明
Prompt 告诉 AI 可以使用以下 4 个工具：
1. **read_paper**：根据 arXiv ID 读取论文完整内容
2. **download_paper**：如果论文还没下载，先用这个工具下载
3. **search_papers**：搜索相关论文来提供上下文背景
4. **list_papers**：查看哪些论文已经下载好了

### 📝 具体工作流程

#### 第一步：准备工作（preparation）
Prompt 明确要求 AI：
1. **先检查论文是否已下载**：使用 `list_papers` 工具查看
2. **如果没下载就下载**：使用 `download_paper` 工具
3. **读取论文内容**：使用 `read_paper` 工具获取完整内容
4. **如果找不到论文**：用 `search_papers` 找相关论文作为替代
5. **如果有相关论文**：也要下载并阅读相关论文的内容

#### 第二步：综合分析（comprehensive-analysis）
Prompt 要求 AI 回答以下 6 个具体问题，形成执行摘要：
- 用 2-3 句话总结这篇论文
- **主要贡献**：这篇论文的主要贡献是什么？
- **解决的问题**：这篇论文要解决的主要问题是什么？
- **使用的方法**：这篇论文使用的主要方法是什么？
- **主要结果**：这篇论文的主要结果是什么？
- **主要结论**：这篇论文的主要结论是什么？

#### 第三步：研究背景分析（research-context）
Prompt 要求分析 4 个方面：
- **研究领域和具体问题**：这个研究属于什么领域？解决什么具体问题？
- **关键的前期方法和局限性**：之前的方法是什么？有什么不足？
- **本文如何推进领域**：这篇论文试图如何推进这个领域？
- **与领域内其他论文的比较**：这篇论文与其他相关论文相比如何？

#### 第四步：方法论分析（methodology-analysis）
Prompt 要求详细分析方法的 6 个层面：
- **方法步骤拆解**：一步一步解释这个方法是怎么做的
- **方法中的关键创新点**：这个方法有哪些创新？
- **理论基础和假设**：理论依据是什么？做了哪些假设？
- **技术实现细节**：具体是怎么实现的？
- **算法复杂度和性能特征**：时间复杂度、空间复杂度、性能如何？
- **可复现性信息**：如果有人想复现这个工作，需要知道什么？

#### 第五步：结果分析（results-analysis）
Prompt 要求分析结果的 5 个方面：
- **实验设置**：用了哪些数据集？哪些基准测试？哪些评估指标？
- **主要实验结果和意义**：实验结果是什么？这些结果的意义是什么？
- **统计有效性和鲁棒性**：结果统计上是否有效？是否鲁棒？
- **结果如何支持或挑战论文的声明**：结果是否证明了论文的声称？
- **与最先进方法的比较**：与 SOTA 方法相比如何？

#### 第六步：实际应用（practical-implications）
Prompt 要求说明 3 点：
- **如何实现或应用**：这个方法实际中怎么用？
- **所需资源和潜在挑战**：需要什么资源？有什么困难？
- **可用的代码、数据集或资源**：论文是否提供了代码、数据等？

#### 第七步：理论意义（theoretical-implications）
Prompt 要求分析 4 个方面：
- **如何推进基本理解**：这项工作如何推进我们对基础理论的理解？
- **引入的新概念或范式**：提出了什么新概念或新范式？
- **对现有理论或假设的挑战**：是否挑战了现有的理论或假设？
- **提出的开放问题**：提出了哪些开放性问题？

#### 第八步：未来方向（future-directions）
Prompt 要求说明 4 点：
- **未来工作可以解决的局限性**：这个方法有哪些局限？未来怎么改进？
- **有前景的后续研究问题**：哪些研究问题值得继续研究？
- **与其他方法整合的潜力**：能否与其他方法结合？
- **这项工作开启的长期研究议程**：这项工作的长期影响是什么？

#### 第九步：更广泛影响（broader-impact）
Prompt 要求分析 3 个方面：
- **社会、伦理或政策影响**：对社会的意义？有什么伦理问题？政策影响？
- **环境或经济考虑**：对环境有什么影响？经济成本如何？
- **潜在的实际应用和时间表**：实际应用场景？什么时候可能应用？

### 💡 额外注意事项
Prompt 还特别提醒 AI：
- 使用 `search_papers` 找相关论文或基于这篇论文的后续工作
- 与其他已分析的论文进行交叉引用
- 使用图表、伪代码等可视化工具来说明关键概念
- 用表格总结关键结果，方便查阅

### 📊 输出要求
- 用清晰的标题结构组织分析
- 保持技术准确性，同时要让读者容易理解
- 在适当的地方加入批判性评估
- 分析要全面但简洁
- **必须批判性评估结果的统计显著性和可复现性**

## 2. `research-discovery` - 研究探索 Prompt

### 📋 Prompt 的核心任务
指导 AI 帮助用户探索和发现某个特定主题的研究。

### 🔧 可用工具说明
告诉 AI 可以使用：
1. **search_papers**：使用高级过滤功能在 arXiv 上搜索论文
2. **download_paper**：下载论文供详细阅读
3. **read_paper**：读取已下载论文的完整内容
4. **list_papers**：查看哪些论文已经下载

### 📝 具体工作流程

#### 第一步：主题探索（topic-exploration）
Prompt 指导 AI：
1. **开始搜索**：使用 `search_papers` 找相关论文
2. **使用合适的类别**：用 arXiv 类别缩小范围（例如 AI/ML 用 cs.AI, cs.LG）
3. **考虑时间限制**：如果提供了时间范围，使用 `date_from/date_to` 参数
4. **审查搜索结果**：识别关键论文、作者和趋势

#### 第二步：论文选择（paper-selection）
Prompt 要求 AI 识别 3 类论文：
- **基础论文**：较早的、被高度引用的奠基性工作
- **最新发展**：展示当前状态的最新论文
- **综述/调研论文**：提供全面概览的综述论文
- 然后下载和阅读关键论文

#### 第三步：知识综合（knowledge-synthesis）
Prompt 要求 AI 做 5 件事：
- **总结主要研究主题和子领域**：这个主题下有哪些主要方向和子方向？
- **识别关键概念、方法论和方法**：这个领域用到了哪些核心概念和方法？
- **注意常见挑战和开放问题**：这个领域有哪些常见困难？还有哪些未解决的问题？
- **突出有影响力的研究者及其贡献**：哪些研究者重要？他们贡献了什么？
- **绘制想法和技术的演变过程**：这个领域的思路和技术是如何演变的？

#### 第四步：定制化（customization）
Prompt 要求根据用户的 **expertise_level** 调整：
- **beginner（初学者）**：
  - 关注基础概念
  - 避免术语，提供解释
  - 让内容容易理解
- **intermediate（中级）**：
  - 平衡概念和技术细节
  - 既有概述也有技术内容
- **expert（专家）**：
  - 深入技术细节
  - 关注高级方法论和细微差别

同时要考虑指定的 **domain（领域）** 的特殊要求。

#### 第五步：输出结构（output-structure）
Prompt 要求按照 7 个部分组织输出：

1. **主题概览（Topic Overview）**
   - 研究领域的简要介绍
   - 为什么这个主题重要/相关
   - 正在解决的关键问题

2. **基础工作（Foundational Work）**
   - 建立该领域的关键早期论文
   - 核心概念和理论

3. **研究现状（Current State of Research）**
   - 最新发展和趋势
   - 正在追求的主要研究方向
   - 值得注意的近期论文及其贡献

4. **关键主题和方法（Key Themes and Approaches）**
   - 常见的方法论和技术
   - 不同的思想流派或方法
   - 方法的比较分析

5. **开放问题和挑战（Open Questions and Challenges）**
   - 未解决的问题
   - 当前方法的局限性
   - 未来的研究方向

6. **推荐论文（Recommended Papers）**
   - 必读的基础论文
   - 重要的近期贡献
   - 用于全面理解的综述/调研论文

7. **入门指南（Getting Started Guide）**
   - 建议的阅读顺序
   - 所需的先修知识和背景
   - 深入探索的下一步

### 💡 最后提醒
- 策略性地使用 `search_papers`，配合合适的查询和过滤器
- 根据需要下载和阅读论文，提供准确、充分的信息
- **根据用户的经验水平和领域背景定制解释**

## 3. `literature-synthesis` - 文献综述 Prompt

### 📋 Prompt 的核心任务
指导 AI 综合多篇学术论文的发现，生成结构化的综述。

### 🔧 可用工具说明
告诉 AI 可以使用：
1. **read_paper**：根据 arXiv ID 读取论文
2. **download_paper**：如果论文还没下载就下载
3. **list_papers**：查看哪些论文可用
4. **search_papers**：如果需要，找额外的相关论文

### 📝 具体工作流程

#### 第一步：准备工作（preparation）
Prompt 指导 AI：
1. 使用 `list_papers` 检查哪些论文已下载
2. 对每个论文 ID，如果没下载就用 `download_paper`，然后用 `read_paper` 获取完整内容
3. 如果指定了 `synthesis_type`，要相应地调整分析方式

#### 第二步：综合方法（synthesis-approaches）
Prompt 根据 **synthesis_type** 参数，要求采用 5 种不同的综合方法：

##### 方法 1：themes（主题综合）
- 按共同主题、话题或研究问题组织论文
- 识别跨论文的重复主题
- 按主题对论文分组
- 比较不同论文如何处理相似主题

##### 方法 2：methods（方法综合）
- 按方法论、技术或方法分类论文
- 按方法论方法对论文进行分类
- 比较不同方法的优点和局限性
- 注意方法论的创新和变化

##### 方法 3：timeline（时间线综合）
- 按时间顺序组织论文以展示演变
- 按发表日期排列论文
- 展示想法和方法如何随时间演变
- 识别关键转折点或突破

##### 方法 4：gaps（空白综合）
- 识别研究空白和未探索的领域
- 注意哪些问题仍未解答
- 识别论文之间的矛盾或不一致之处
- 突出研究覆盖有限的领域
- 建议潜在的研究方向

##### 方法 5：comprehensive（全面综合）
- 结合主题、方法和时间线分析
- 创建研究领域的全面概览
- 包括上述所有方面，再加上批判性分析

#### 第三步：输出结构（output-structure）
Prompt 要求按照 7 个部分组织输出（根据 synthesis_type 调整）：

1. **执行摘要（Executive Summary）**
   - 正在综合的论文概览
   - 主要发现和结论
   - 采用的综合方法

2. **综合分析（Synthesis Analysis）**
   - **如果是 themes**：主题组织和比较
   - **如果是 methods**：方法论分类和评估
   - **如果是 timeline**：时间演变和进展
   - **如果是 gaps**：识别的空白和研究机会
   - **如果是 comprehensive**：多维度综合

3. **跨论文的主要发现（Key Findings Across Papers）**
   - 共同发现和共识点
   - 不同观点或矛盾
   - 个别论文的突出贡献

4. **比较分析（Comparative Analysis）**
   - 不同方法的优点和局限性
   - 论文之间的关系和依赖
   - 论文如何相互建立或挑战

5. **综合结论（Synthesis Conclusions）**
   - 识别的整体模式和趋势
   - 从综合中得出的关键见解
   - 对研究领域的影响

6. **研究空白和未来方向（Research Gaps and Future Directions）**
   - 未解答的问题
   - 需要更多研究的领域
   - 有前景的未来方向

7. **参考文献和论文详情（References and Paper Details）**
   - 每篇论文的关键贡献摘要
   - 每篇论文如何融入更广阔的图景

### 💡 最后提醒
- **仔细阅读所有指定的论文**
- **提供超出简单总结的细致综合**——识别模式、关系、矛盾和空白
- **使用 synthesis_type 来适当地聚焦分析**

## 4. `research-question` - 研究问题生成 Prompt

### 📋 Prompt 的核心任务
指导 AI 基于现有文献提出研究问题。

### 🔧 可用工具说明
告诉 AI 可以使用：
1. **read_paper**：根据 arXiv ID 读取论文
2. **download_paper**：如果论文还没下载就下载
3. **list_papers**：查看哪些论文可用
4. **search_papers**：如果需要，找额外的相关论文

### 📝 具体工作流程

#### 第一步：文献综述（literature-review）
Prompt 指导 AI：
1. 使用 `list_papers` 检查哪些论文可用
2. 对每个论文 ID，如果没下载就用 `download_paper`，然后用 `read_paper` 获取完整内容
3. **理解每篇论文的研究背景、方法、发现和局限性**
4. **识别论文之间的关系以及它们如何与给定主题关联**

#### 第二步：空白分析（gap-analysis）
Prompt 要求 AI 进行 6 项分析：
- **识别论文解决了什么问题**：这些论文都回答了哪些问题？
- **注意哪些问题仍未解答或只部分解答**：还有什么没搞清楚？
- **找出论文之间的矛盾或不一致**：论文之间有没有冲突的结论？
- **识别方法论的局限性或空白**：方法上有什么不足？
- **注意需要更多研究的领域**：哪些方向还需要更多工作？
- **考虑研究主题如何与更广泛的领域问题相关**：这个问题在整个领域中的地位如何？

#### 第三步：问题制定（question-formulation）
Prompt 要求制定研究问题时，必须满足 5 个条件：
1. **建立在现有工作基础上**：不能凭空而来
2. **解决已识别的空白**：要填补研究的空缺
3. **新颖且有意义**：要有创新性和重要性
4. **可行研究**：能够实际调查
5. **对研究领域有贡献**：要推动领域发展

Prompt 还要求考虑 6 种不同类型的研究问题：
- **描述性问题**：X 是什么？（What is X?）
- **比较性问题**：X 与 Y 相比如何？（How does X compare to Y?）
- **解释性问题**：为什么 X 会发生？（Why does X occur?）
- **探索性问题**：X 的特征是什么？（What are the characteristics of X?）
- **评估性问题**：X 的有效性如何？（How effective is X?）
- **方法论问题**：我们如何更好地测量/研究 X？（How can we better measure/study X?）

并且确保问题**具体、清晰、可研究**。

#### 第四步：背景化（contextualization）
Prompt 要求：
- **将问题与更广泛的研究领域联系起来**：这个问题在整个领域中的位置
- **考虑领域特定的要求和约束**：这个领域有什么特殊要求？
- **在研究领域中定位问题**：这个问题在研究版图上的位置
- **解释为什么这些问题重要**：为什么要研究这些问题？

#### 第五步：输出结构（output-structure）
Prompt 要求按照 6 个部分组织输出：

1. **研究背景（Research Context）**
   - 研究主题的摘要
   - 审查的现有文献概览
   - 关键发现和当前知识状态

2. **文献空白分析（Literature Gap Analysis）**
   - 现有论文已建立的内容
   - 仍未知或不确定的内容
   - 当前研究的局限性和空白
   - 未解决的矛盾或争论

3. **制定的研究问题（Formulated Research Questions）**
   - 主要研究问题
   - 次要或支持性问题
   - **对于每个问题，必须包括**：
     * 问题本身（清晰陈述）
     * 为什么这个问题重要
     * 它如何解决已识别的空白
     * 它与现有文献的关系
     * 解决它的潜在方法

4. **问题优先级（Question Prioritization）**
   - 最需要首先解决的关键问题
   - 会产生最大影响的问题
   - 可行性考虑
   - 问题之间的相互依赖

5. **研究方向（Research Directions）**
   - 解决这些问题的潜在方法论
   - 所需资源和考虑因素
   - 预期结果和贡献
   - 解决这些问题如何推进该领域

6. **相关研究机会（Related Research Opportunities）**
   - 分析过程中出现的其他问题
   - 未来调查的问题
   - 与其他研究领域的联系

### 💡 最后提醒
- **彻底分析提供的论文和研究主题**
- **制定的研究问题要既建立在现有文献基础上又具有创新性**
- **问题要具体、有意义且可行**
- **在适用时考虑领域特定的背景**

## 📊 四个 Prompts 的对比总结

| Prompt | 输入 | 核心任务 | 输出类型 |
|--------|------|---------|---------|
| **deep-paper-analysis** | 1篇论文ID | 深度分析单篇论文的所有方面 | 详细的分析报告 |
| **research-discovery** | 1个研究主题 | 探索主题，找到相关论文和方向 | 研究概览和推荐论文列表 |
| **literature-synthesis** | 多篇论文ID | 综合多篇论文的发现 | 结构化的文献综述 |
| **research-question** | 多篇论文ID + 主题 | 基于文献提出新问题 | 研究问题清单和方向 |

---

## 🎯 使用建议

- **想深入了解一篇论文** → 用 `deep-paper-analysis`
- **想探索一个新领域** → 用 `research-discovery`
- **想整合多篇论文的发现** → 用 `literature-synthesis`
- **想找到新的研究方向** → 用 `research-question`
